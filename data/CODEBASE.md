## Runtime flow

1. FastAPI receives `POST /api/v1/chat {message, session_id}`
2. `SessionStore` loads prior conversation turns for this session
3. `SuperAgentOrchestrator.run()` injects today's date + SOUL + MEMORY + this file
4. `classify_intent()` routes: SYMBOLIC → SymPy codegen loop, NEURAL → Gemini
5. Current-events keywords → `generate_with_search()` (Google Search grounding)
6. Multi-turn history passed as `contents=[Content...]` to Gemini SDK
7. Result stored in HDC associative memory (`data/hdc_memory.json`)
8. Session turn saved to `data/sessions/<id>.json`

## Key capabilities

- **Self-aware**: knows its own file structure (this document)
- **Multi-turn**: per-session conversation history backed to JSON
- **Symbolic math**: Gemini generates SymPy → executed deterministically
- **Search grounding**: current events queries use Google Search tool
- **HDC memory**: 10 000-dim hypervectors for associative retrieval
- **SICA loop**: self-assessment → plan JSON → AST check → git commit
- **Heartbeat**: APScheduler reads HEARTBEAT.md every 30 min, updates MEMORY.md
- **Model fallback**: 429/404 from primary model → tries fallback chain
